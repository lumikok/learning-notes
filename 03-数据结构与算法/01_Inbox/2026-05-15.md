## 从迷宫说起：DFS 到底在干什么

想象你走进一个迷宫，没有地图，出口在未知的地方。

你会怎么做？最自然的策略就是：

👉随便挑一条路，**一直往前走**，不回头。
👉 如果走到死胡同，就**退回到上一个岔路口**，换一条没试过的路继续走。  
👉 重复这个过程，直到找到出口。

这就是深度优先搜索 （DFS）的核心思想：**“一条道走到黑 + 走不通就回溯”**。

DFS 本质上是一种**系统性的穷举搜索**，适用于树、图的遍历，以及所有需要枚举可能性的问题（如全排列、子集、迷宫路径）。

## 用“树”来理解 DFS

你可以用**树**来直观理解 DFS：

- **节点**：代表一个“状态”（比如你在迷宫中的位置、当前已选的数字）。
- **边**：代表一个“操作”（比如向前走一步、选下一个数字）。

DFS 的过程：

- 从根出发（**最初状态**）。
- 一直往一个分支钻，走到尽头。
- 走到底后，退回去，再钻另一个分支

### 前序遍历示例（根 → 左 → 右）

```cpp
#include <iostream>
using namespace std;

struct TreeNode {
	int val;
	TreeNode *left;
	TreeNode *right;
	TreeNode(int x): val(x),left(nullptr),right(nullptr) { }
};

void dfs(TreeNode *node) {
	if(node == nullptr) return;

	cout << node->val << " "; 	// 访问当前节点
	dfs(node->left);          	// 递归遍历左子树
	dfs(node->right);	        // 递归遍历右子树
}

int main() {
    TreeNode *root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    dfs(root); // 输出 1 2 3
    return 0;
}
```

**模拟过程**：从根节点 1 开始，先输出 1，然后进入左子树 2，输出 2，左边没有子节点，回溯到 1，再进入右子树 3，输出 3。

## 图上 DFS：为什么需要 `visited`？

图可能存在环，也可能有多条路径到达同一节点。若不记录已访问节点，DFS 会陷入无限循环或重复处理。

解决方法：**用 visited 数组记录已访问节点**。

```cpp
#include <iostream>
#include <vector>
using namespace std;

void dfs(int u,vector<int>& visited,const vector<vector<int>>& graph) {
    if(visited[u]) return;
    visited[u] = 1;
    cout << u << " ";
    for(int v : graph[u]) {
        if(!visited[v])
        dfs(v,visited,graph);
    }
}

int main() {
    vector<vector<int>> graph = {{1,2},{0,3,4},{0},{1},{1}};
    vector<int> visited(graph.size(),0);

    dfs(0,visited,graph); // 输出 0 1 3 4 2 （顺序依据访问顺序）
    return 0;
}
```

**需要 visited 的场合**：图中存在环，或者即使无环但有多条路径可达同一节点。
树是特例：树的 DFS 天然不会重复访问，因为父节点到子节点是单向的，且无环。

## DFS 的两种实现方式：递归 VS 迭代

### 1️⃣递归实现（最常用，代码简洁）

上述代码示例都是用的递归实现，下面列出其优缺点：

- **优点**：代码逻辑与算法定义一致，易于理解。
- **缺点**：递归深度受系统栈限制（通常约 1 e 5 层），过深会导致栈溢出。

### 2️⃣迭代实现（用栈模拟递归）

**为什么是栈？**

DFS 要求“后探索的节点优先回溯”，即最后压入的节点最先被取出，这正是栈（LIFO）的行为。

因此，可用 `std:stack` 来手动模拟递归过程：

```cpp
#include <stack>
void dfs_interative(int start,const vector<vector<int>>& graph) {
    vector<int> visited(graph.size(),0);
    stack<int> s;
    s.push(start);
    while(!s.empty()) {
        int u = s.top();
        s.pop();
        if(visited[u]) continue;
        visited[u] = 1;

        cout << u << " ";
        
        // 注意：栈是后进先出，故和递归的输出顺序相反
        for (int v : graph[u]) {
            if(!visited[v]) {
                s.push(v);
            }
        }
    }
}
```

一句话：**递归靠系统栈，迭代靠手动栈。**

## 经典回溯问题：全排列

**问题**：给定不含重复数字的数组 `nums`，输出所有排列。

**示例**：`nums = [1,2,3]` 输出 6 种排列。

### 为什么需要 `used` 数组？

显而易见，我们不能选择已被使用的数字，因此需要定义一个 `used` 数组来标记数字是否已使用。**这是回溯问题的基础设施**。

### 代码实现

```cpp
void dfs(vector<int>& nums, vector<bool>& used, vector<int>& path) {
    if (path.size() == nums.size()) {
        for (int x : path) cout << x << " ";
        cout << endl;
        return;
    }
    for (int i = 0; i < nums.size(); i++) {
        if (!used[i]) {
            used[i] = true;
            path.push_back(nums[i]);
            dfs(nums, used, path);
            path.pop_back();   // 回溯：撤销选择
            used[i] = false;   // 回溯：释放标记
        }
    }
}
```

**关键点**：

- 递归前：做选择（标记 used、加入路径）。
- 递归后：撤销选择（弹出路径、清除 used）—— **回溯常用操作**。

## 复杂度分析

| 场景            | 时间复杂度      | 空间复杂度（不含输入）            |
| ------------- | ---------- | ---------------------- |
| 树遍历（每个节点访问一次） | O(N)，N为节点数 | 递归栈O(树高)               |
| 图遍历（邻接表）      | O(V + E)   | visited O(V) + 递归栈O(V) |
| 全排列           | O(n × n!)  | used O(n) + 递归栈O(n)    |

## 注意事项

| 错误写法                  | 后果         | 正确做法                                   |
| --------------------- | ---------- | -------------------------------------- |
| 图上 DFS 未标记 visited    | 无限循环 / 栈溢出 | 进入节点时立即标记 `visited[u] = 1`             |
| 回溯后忘记恢复 used 或 path   | 排列/子集结果错误  | 递归后 `used[i]=false`, `path.pop_back()` |
| 递归函数参数传值（非引用）         | 大量拷贝，效率低   | 传引用或使用全局变量                             |
| 递归深度过深（> 1e5）         | 栈溢出崩溃      | 改为迭代栈实现                                |
| 迭代 DFS 压栈时未检查 visited | 重复访问，效率低   | 压栈前检查 `if (!visited[v])` 或出栈时检查        |

## 总结

- **DFS 核心**：深入到底 + 回溯。
- **DFS 的本质**：用栈实现。
- **树上的 DFS**：不需要 `visited`（无环 + 唯一路径）。
- **图上的 DFS**：必须 `visited`（防环 / 防重复访问）。
- **两种实现**：递归（简洁）和迭代（防栈溢出）。
- **全排列**：是 DFS 回溯的一个典型应用。
